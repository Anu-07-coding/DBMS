Data

Data are the collection of raw facts and figure which are unorganized, uninterpreted, and isolated, which doesn’t give any particular sense. They are randomly gathered. So that they can be processed to obtain the desired result.

Data types

Data types are the kind of data that can be stored in a database column, dictating how the data is stored and the operations that can be performed on it. Common categories include numeric, character/string, date and time, binary, and boolean data types. Specialized types like XML, JSON, and spatial data types are also used.
  1. Numeric Data Types
    • Integer: Used for whole numbers (e.g., INT, TINYINT, BIGINT)
    • Floating-point: Used for numbers with decimal points (e.g., FLOAT, REAL)
  2. Character and String Data Types
    • CHAR: Fixed-length character strings (e.g., CHAR(10) for a string of exactly 10 characters)
    • VARCHAR: Variable-length character strings (e.g., VARCHAR(255) for a string up to 255 characters)
    • TEXT: Used for storing large text values
    • NCHAR/NVARCHAR/NTEXT: Unicode character string datatypes for storing characters from different languages
  3. Date and Time Data Types
    • DATE: Stores dates (e.g., 2025-06-22)
    • TIME: Stores time (e.g., 14:30:00)
    • DATETIME: Stores both date and time
  4. Binary Data Types
    • BINARY: Fixed-length binary data
    • VARBINARY: Variable-length binary data
  5. Boolean Data Type
    • BOOLEAN: Stores true or false values
  6. Special Data Types
    • JSON: Stores data in JSON (JavaScript Object Notation) format
    • XML: Stores data in XML (Extensible Markup Language) format
    • Spatial data types: Used for storing geographic data
    • BLOB/CLOB: Used for storing large binary objects (BLOB) and character large objects (CLOB)
    • CURSOR: Used to store a reference to a result set
    • TABLE: Used to store a table structure
  7. User-Defined Data Types
    • Users can create their own data types based on existing ones

    Database Management System (DBMS)

    A Database Management System (DBMS) is a software solution designed to efficiently manage, organize, and retrieve data in a structured manner.DBMS is a computerized record keeping system. It is a software that defines, manipulates and manage the database. It allows to access the file, update the record and retrieve data as requested. In other word, DBMS is defined as the collection of interrelated data and set of programs to access there data. The collection of data is usually a database which contains the information about any particular organization. The primary goal of DBMS is to provide an effective and efficient environment for both data retrieval and storing data in database

    • Advantages of DBMS
      1. Sharing data
      2. Reduce data redundancy
      3. Data backup and recovery
      4. Data security
      5. Multiple user interface
      6. Data integrity

    • Disadvantages of DBMS
      1. Expensive
      2. Changing technology
      3. Backup data needed

      Types of Database

      1. Hierarchical database model
        2.  It is one of the oldest type of database model. In this model data are represented in the forms of record, each record has multiple field or attributes. All records are arranged in database as tree like structure. The relationship between the records is called parent child relationship in which one child record relates to only a single parent i.e child posses property only property of a single parent. Here child are restricted to use the property of a parent to whom it doesn't belong.
      2. Network database model
        3.  This network model replace hierarchical model due to some limitation on the model. Suppose an employee relates to two different department then hierarchical model cannot be able to arrange record in proper place. So, network database model was emerged to arrange non-hierarchical database. The structure of database is more like graph rather than tree structure. A network model consists of collection of record which are interrelated to each other with the help of relationship. Each records have multiple fields and each field has only one data value. In this type of model a parent may have multiple child, as well as child can have multiple parent.
      3. Relational database model
        4.  In relational database model, the data are organized into tables which contains multiple row and columns. These tables are called relations. A row in a table represents a relationship among a set of values. Since, a table is collection of such relationship. It is generally referred to the mathematical term relations from which the relational database model derives its name. It is also known as RDBMS.
      4. Centralized database
        5.  It is a simple type which works on client server basis. In this type clients or user are directly connected to the centrally totally located server. This server hosts the data of its client or user and helps them to store and retrieve data as requirement. This type of system are used in small scale industries which doesn't have to deal with large volume of data and user. Centralized database runs on single computer which may have single or multiple user. Since database is centralized, security is not a crucial part here. The maintenance of database is easier because of data are centrally stored. This type of system denote allow unauthorized person to access data.
      5. Distributed database
        6.  This type of database system are complex in structure, instead of storing and retrieving data from centrally located server, it uses several numbers of database and server randomly located at different place. It is the collection of multiple logically interrelated database which are distributed in many geographical location. Since server are located at different locations user can experience a good speed of bandwidth. Similarly, back up and recovery process is lot more easier there, which makes data more secured. This type of system is used by large organization who has to deal with large volume of data and user all around the world. Since it is distributed in nature there may arises security issue and are costly to maintain and operate. Simply, distributed database system are the collection of several number of centralized database system in different locations.

        • Common Functions that a DBMS performs:

        1. Administration tasks
          2. A DBMS supports many typical database administration tasks, including change management, performance monitoring and tuning, security, and backup and recovery. Most database management systems are also responsible for automated rollbacks and restarts as well as logging and auditing of activity in databases and the applications that access them.
        2. Storage
          3. A DBMS provides efficient data storage and retrieval by ensuring that data is stored in tables, rows and columns.
        3. Concurrency control
          4. In environments where multiple users access and modify the database simultaneously, a DBMS guarantees controlled transaction execution to prevent data corruption or inconsistency.
        4. Centralized view
          5. A DBMS provides a centralized view of data that multiple users can access from multiple locations in a controlled manner. A DBMS can limit what data end users see and how they view the data, providing many views of a single database schema. End users and software programs are free from having to understand where the data is physically located or on what type of storage medium it resides because the DBMS handles all requests.
        5. Data manipulation
          6. A DBMS ensures data integrity and consistency by letting users insert, update, delete and modify data inside a database.
        6. Data independence
          7. A DBMS offers both logical and physical data independence to protect users and applications from having to know where data is stored or from being concerned about changes to the physical structure of data. As long as programs use the application programming interface (API) for the database that the DBMS provides, developers won't have to modify programs just because changes have been made to the database.
        7. Backup and recovery
          8. A DBMS facilitates backup and recovery options by creating backup copies so that data can be restored to a consistent state. This protects against data loss due to hardware failures, software errors or other unforeseen events. In a relational database management system (RDBMS) -- the most widely used type of DBMS -- the API is structured query language (SQL), a standard programming language for defining, protecting and accessing data.

        • Components of DBMS

        1. Storage engine
          2. This basic element of a DBMS is used to store data. The DBMS must interface with a file system at the operating system (OS) level to store data. It can use additional components to store data or interface with the actual data at the file system level.
        2. Database access language
          3. The DBMS must also provide an API to access the data, typically in the form of a database access language that can be used to modify data but also create database objects and secure and authorize access to the data. SQL is an example of a database access language and encompasses several sets of commands, including data control language for authorizing data access, data definition language for defining database structures and data manipulation language for reading and modifying data.
        3. Optimization engine
          4. A DBMS can also provide an optimization engine that's used to parse database access language requests and turn them into actionable commands for accessing and modifying data.
        4. Query processor
          5. After a query is optimized, the DBMS must provide a way to run the query and return results.
        5. Lock manager
          6. This crucial component of the DBMS manages concurrent access to the same data. Locks are required to ensure multiple users aren't trying to modify the same data simultaneously.
        6. Data utilities
          7. A DBMS also provides a set of utilities for managing and controlling database activities. Examples of database utilities include reorganization, RUNSTATS, backup and copy, recover, integrity check, load data, unload data and repair database.
        7. Reporting and monitoring tools
          8. Most DBMSes are integrated with reporting and monitoring tools to offer enhanced functionality for managing and analyzing data. Reporting tools generate reports, whereas monitoring tools track various database metrics, such as resource consumption and user activity.


        • Structure Query Language (SQL)

        SQL stands for Structured Query Language. It is an international standard data base query language for accessing and managing data in the database. SQL was introduced and developed by IBM in early 1970’s. It was able to control relational database. SQL is not a complete programming language. It is only used for communicating with database. SQL has statement for data definition (DDL), data manipulation (DML) and data control (DCL). A query is a request to a DBMS for the retrieval, modification, insertion and deletion of the data from database. SQL is made of three sub languages: DDL, DML and DCL
        1. DDL (Data Definition Language)
          2. DDL is used by the database designer and programmers to specify the content and the structure of table. It is used to define the physical characteristics of record. It includes commands that manipulate the structure of object such as tables: For eg, to create table
          Syntax:
          CREATE TABLE table_name (field_name1 data_type1 field_name2 data_type2 ………);
          CREATE TABLE Student (SN Number Fname text);
        2. DML (Data Manipulation Language)
          3. DML is related with manipulation of records such as retrieval, sorting, display and deletion of records or data. It helps user to use query and display report of the table. It provide technique for processing the database. It includes commands like insert, delete, select, and update to manipulate the information stored in the database.
          Syntax:
          INSERT INTO table_name VALUES (list of values);
          INSERT INTO student VALUES (1 RAM);
        3. DCL (Data Control Language)
          4. All provides additional feature for security of table and database. It includes commands for controlling data and access to the database. Some of the example of this command are grant, commit, etc

          • CRUD Operations

          CRUD is an acronym for CREATE, READ(SELECT), UPDATE, and DELETE statements in SQL Server. CRUD in database terms can be mentioned as Data Manipulation Language (DML) Statements as well. Data Manipulation Language is used to manage or manipulate the data present inside database Tables. Even though CRUD operations are supported on Database object View, Views have several restrictions and hence in this article, we can test the CRUD operations on Tables.
          1. Create (INSERT)
            2. The Create operation is used to insert new records into a database table. It uses the SQL INSERT INTO statement to add a new row of data.
          2. Read (SELECT)
            3. The Read operation is used to retrieve or view existing data from the database. It uses the SELECT statement, which can display specific columns or all data.
          3. Update (UPDATE)
            4. The Update operation is used to modify or change existing records in a table. The UPDATE statement is used along with the SET and WHERE clauses to target specific records.
          4. Delete (DELETE)
            5. The Delete operation is used to remove existing records from a table. It uses the DELETE FROM statement along with a condition to specify which record(s) to delete.

          Keys in DBMS
          Keys are essential concepts in relational databases. They help uniquely identify rows (records) in a table and establish relationships between tables. Keys also help maintain data integrity and eliminate redundancy.

          • Types of Keys in DBMS

          Primary Key: A primary key is a column or a set of columns that uniquely identifies each row in a table.It cannot contain NULL values.A table can have only one primary key.

          Candidate Key: A candidate key is any column (or combination) that can qualify as a primary key.There can be multiple candidate keys in a table.One of them is chosen as the primary key.

          Alternate Key: When a candidate key is not chosen as the primary key, it is called an alternate key.It is still capable of uniquely identifying records.

          Composite Key: A composite key is made up of two or more columns that together uniquely identify a row.It is used when a single column is not sufficient to uniquely identify records.

          Foreign Key: A foreign key is a column in one table that refers to the primary key of another table.It is used to link two tables and enforce referential integrity.

          Constraints in Database
          Constraints in a database are rules applied to table columns to ensure the accuracy, consistency, and integrity of data. The NOT NULL constraint ensures that a column cannot have empty or null values, enforcing that data must be provided. The UNIQUE constraint guarantees that all values in a column are different, allowing only one occurrence of each value. A PRIMARY KEY uniquely identifies each record in a table by combining the features of NOT NULL and UNIQUE, and there can only be one primary key per table. The FOREIGN KEY constraint is used to maintain referential integrity between two tables by linking a column in one table to the primary key of another. The CHECK constraint restricts the values that can be entered into a column by specifying a condition that must be met, such as limiting marks to a range between 0 and 100. Finally, the DEFAULT constraint assigns a default value to a column when no value is provided during data entry. Together, these constraints help maintain reliable and valid data within relational databases.
          • Anomalies in Database
          Anomalies are problems or inconsistencies that occur in a database table when data is poorly organized, especially in un-normalized or redundant tables. These issues can lead to data duplication, loss, or inconsistency, and they are typically addressed by applying normalization techniques.
          • Types of Anomalies
          1. Insertion Anomaly
            2. It occurs when we cannot insert data into a table without the presence of other data.This usually happens in a table where multiple entities are stored together.
          2. Update Anomaly
            3. It happens when data is duplicated in multiple rows, and updating it in one place requires updating it in all other places.If you forget to update one instance, it leads to inconsistency.
          3. Deletion Anomaly
            4. It occurs when deleting one piece of data unintentionally removes other useful data.This usually happens when unrelated data is stored together.

            • Normalization in Database
            Normalization is the process of organizing data in a database to reduce redundancy (repeated data) and improve data integrity. It involves dividing a large table into smaller, related tables and linking them using keys. The goal is to make the database efficient, consistent, and easy to maintain.

            • Why is Normalization Important?

            Eliminates data redundancy (duplicate data)
            Prevents anomalies (insertion, update, and deletion issues)
            Ensures data consistency and accuracy
            Improves data organization and scalability

            • Types of Normalization

            1. First Normal Form (1NF)
              2. A table is in 1NF if:
              All columns contain atomic (indivisible) values
              There are no repeating groups or arrays
              It ensures that each field contains only one value
            2. Second Normal Form (2NF)
              3. A table is in 2NF if:
              It is already in 1NF
              Every non-key column is fully dependent on the entire primary key
              Mainly applies to tables with composite keys
            3. Third Normal Form (3NF)
              4. A table is in 3NF if:
              It is already in 2NF
              There is no transitive dependency (non-key column depends on another non-key column)
              It ensures that every column is only dependent on the primary key

              • Database security

              Database security refers to the protection of the database against unauthorized access, misuse, or corruption. It ensures that only authorized users can access or modify the data, while confidentiality, integrity, and availability of the data are maintained.

              Objectives of Database Security:
              1. Prevent unauthorized access
              2. Protect sensitive data from theft or leaks
              3. Ensure data integrity (no unauthorized changes)
              4. Ensure data availability (data is accessible to valid users)
              5. Prevent data loss or corruption

                6. Common Security Threats:
                1. Unauthorized access (hacking or misuse of credentials)
                2. SQL injection attacks
                3. Data breaches or leaks
                4. Physical theft or damage
                5. Malware or viruses
                6. Human errors (accidental deletion or updates)

                  7. Techniques to Secure a Database:
                  1. User Authentication
                  2. Authorization and Access Control
                  3. Data Encryption
                  4. Backups and Recovery
                  5. Database Auditing
                  6. Firewall and Network Security
